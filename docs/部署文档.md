# SQL Monitor Spring Boot Starter 部署文档

## 1. 项目概述

SQL Monitor Spring Boot Starter 是一个企业级的SQL监控组件，基于Spring Boot 3开发，提供慢SQL监控、敏感信息过滤、MDC监控等功能，帮助企业及时发现和解决SQL执行问题，提高系统的性能和可靠性。

## 2. 环境要求

### 2.1 硬件要求

| 配置 | 最低要求 | 推荐要求 |
|------|---------|----------|
| CPU | 2核 | 4核及以上 |
| 内存 | 4GB | 8GB及以上 |
| 磁盘 | 50GB | 100GB及以上 |

### 2.2 软件要求

| 软件 | 版本 | 说明 |
|------|------|------|
| JDK | 17+ | 必须使用Java 17或更高版本 |
| Spring Boot | 3.0+ | 基于Spring Boot 3开发 |
| MyBatis | 3.5+ | 用于SQL拦截 |
| Maven | 3.6+ | 用于构建项目 |
| 数据库 | MySQL/Oracle/PostgreSQL等 | 支持多种数据库 |

## 3. 构建与部署

### 3.1 构建项目

1. **克隆代码**

```bash
git clone <repository-url>
cd sql-monitor-spring-boot-starter
```

2. **构建项目**

```bash
mvn clean package -DskipTests
```

3. **安装到本地仓库**

```bash
mvn install -DskipTests
```

### 3.2 发布到远程仓库

1. **配置Maven settings.xml**

在`~/.m2/settings.xml`文件中添加仓库配置：

```xml
<servers>
    <server>
        <id>nexus-releases</id>
        <username>your-username</username>
        <password>your-password</password>
    </server>
    <server>
        <id>nexus-snapshots</id>
        <username>your-username</username>
        <password>your-password</password>
    </server>
</servers>
```

2. **发布到远程仓库**

```bash
mvn deploy -DskipTests
```

## 4. 集成到企业应用

### 4.1 添加依赖

在企业应用的`pom.xml`文件中添加依赖：

```xml
<dependency>
    <groupId>com.hyw</groupId>
    <artifactId>sql-monitor-spring-boot-starter</artifactId>
    <version>3.0.0</version>
</dependency>
```

### 4.2 配置应用

在应用的`application.yml`或`application.properties`文件中添加配置：

```yaml
# 慢SQL监控组件配置
hyw:
  sql:
    monitor:
      enabled: true
      slow-threshold: 800        # 慢SQL阈值(ms)
      critical-threshold: 3000   # 严重慢SQL阈值(ms)
      log-enabled: true          # 是否记录采样日志
      sample-rate: 0.05          # 采样率 5%
      max-sql-length: 3000       # SQL最大截断长度
      max-cache-size: 2000       # 缓存最大数量
      db-type-cache-seconds: 60  # 数据库类型缓存时间

      # 脱敏配置
      sensitive:
        enabled: true
        mask-char: "*"
        mask-length: 4
        sensitive-fields:
          - password
          - pwd
          - secret
          - token
          - id_card
          - phone
          - mobile
          - email
          - bank_card
        sensitive-tables:
          - user
          - account
          - payment
          - customer

      # 线程池配置
      pool:
        core-size: 4
        max-size: 8
        queue-capacity: 2000
        keep-alive-seconds: 60

      # MDC链路追踪配置
      mdc:
        enabled: true
        monitor-rate: 0.01        # MDC监控采样率
        alert-threshold: 99.5      # 告警阈值
        mdc-keys:
          - traceId
          - spanId
          - userId
          - requestId
          - clientIp
          - serverIp

      # Micrometer 指标（默认关闭，按需开启）
      metrics:
        enabled: false
        include-sql-id: false

# 指标配置
management:
  endpoints:
    web:
      exposure:
        include: prometheus,health,info
  metrics:
    tags:
      application: ${spring.application.name}

# 日志级别配置
logging:
  level:
    com.hyw.boot: INFO
    com.hyw.boot.monitor: INFO  # 生产环境建议 INFO
    com.baomidou.mybatisplus: INFO
    # SQL 日志级别（生产环境建议关闭）
    com.hyw.mapper: INFO
  pattern:
    console: "%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] [%X{traceId}] %-5level %logger{36} - %msg%n"
    file: "%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] [%X{traceId}] %-5level %logger{36} - %msg%n"
  file:
    name: logs/sql-monitor.log
    max-size: 100MB
    max-history: 30
```

## 5. 容器化部署

### 5.1 Docker构建

1. **创建Dockerfile**

```dockerfile
FROM openjdk:17-jdk-slim

WORKDIR /app

COPY target/your-application.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar"]
```

2. **构建Docker镜像**

```bash
docker build -t your-application:latest .
```

3. **运行Docker容器**

```bash
docker run -d -p 8080:8080 --name your-application your-application:latest
```

### 5.2 Kubernetes部署

1. **创建Deployment**

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: your-application
  namespace: default
spec:
  replicas: 3
  selector:
    matchLabels:
      app: your-application
  template:
    metadata:
      labels:
        app: your-application
    spec:
      containers:
      - name: your-application
        image: your-application:latest
        ports:
        - containerPort: 8080
        resources:
          limits:
            cpu: "2"
            memory: "4Gi"
          requests:
            cpu: "1"
            memory: "2Gi"
        env:
        - name: SPRING_PROFILES_ACTIVE
          value: "prod"
```

2. **创建Service**

```yaml
apiVersion: v1
kind: Service
metadata:
  name: your-application
  namespace: default
spec:
  selector:
    app: your-application
  ports:
  - port: 80
    targetPort: 8080
  type: LoadBalancer
```

3. **部署到Kubernetes**

```bash
kubectl apply -f deployment.yaml
kubectl apply -f service.yaml
```

## 6. 监控与告警

### 6.1 Prometheus集成

1. **添加Prometheus依赖**

```xml
<dependency>
    <groupId>io.micrometer</groupId>
    <artifactId>micrometer-registry-prometheus</artifactId>
</dependency>
```

2. **配置Prometheus**

在`application.yml`文件中添加配置：

```yaml
management:
  endpoints:
    web:
      exposure:
        include: prometheus,health,info
  metrics:
    tags:
      application: ${spring.application.name}
```

3. **Prometheus配置**

在Prometheus配置文件中添加：

```yaml
scrape_configs:
  - job_name: 'your-application'
    metrics_path: '/actuator/prometheus'
    static_configs:
      - targets: ['your-application:8080']
```

### 6.2 Grafana仪表盘

1. **导入仪表盘**

在Grafana中导入SQL监控仪表盘，或创建自定义仪表盘。

仓库已提供可直接导入的 Grafana Dashboard：

- `grafana/sql-monitor-dashboard.json`

导入方式：Grafana -> Dashboards -> New -> Import -> 上传该 JSON 文件。

### 6.4 Actuator 健康检查（可选）

业务方引入 `spring-boot-starter-actuator` 后，可通过 `/actuator/health` 查看 SQL Monitor 健康状态（熔断状态、连续失败次数、线程池状态等）。

2. **关键指标监控**

- SQL执行时间分布
- 慢SQL数量
- 严重慢SQL数量
- MDC丢失率
- 线程池状态

### 6.3 告警配置

1. **Prometheus告警规则**

```yaml
groups:
- name: sql-monitor-alerts
  rules:
  - alert: SlowSqlRateHigh
    expr: sum(increase(sql_slow_count_total[5m])) / sum(increase(sql_execution_time_seconds_count[5m])) > 0.05
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "慢SQL率过高"
      description: "应用 {{ $labels.application }} 的慢SQL率超过5%"
  - alert: CriticalSlowSqlDetected
    expr: sum(increase(sql_slow_count_total{sqlId=~"critical.*"}[5m])) > 0
    for: 1m
    labels:
      severity: critical
    annotations:
      summary: "检测到严重慢SQL"
      description: "应用 {{ $labels.application }} 检测到严重慢SQL"
  - alert: MdcLossRateHigh
    expr: sum(increase(mdc_miss_count_total[5m])) / sum(increase(mdc_total_count_total[5m])) > 0.1
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "MDC丢失率过高"
      description: "应用 {{ $labels.application }} 的MDC丢失率超过10%"
```

2. **告警渠道配置**

配置Prometheus Alertmanager，将告警发送到邮件、Slack、钉钉等渠道。

## 7. 性能优化

### 7.1 配置优化

| 场景 | 配置建议 | 说明 |
|------|---------|------|
| 高并发场景 | 增加线程池核心线程数和最大线程数 | 提高异步处理能力 |
| 大数据量场景 | 增加缓存大小，调整过期时间 | 减少缓存 miss |
| 生产环境 | 降低采样率，关闭详细日志 | 减少日志输出 |
| 内存受限场景 | 减少缓存大小，调整线程池配置 | 降低内存使用 |

### 7.2 JVM优化

```bash
java -Xms4g -Xmx8g -XX:MetaspaceSize=256m -XX:MaxMetaspaceSize=512m -XX:+UseG1GC -XX:MaxGCPauseMillis=200 -jar app.jar
```

### 7.3 数据库优化

- 确保数据库索引正确创建
- 优化SQL语句，避免全表扫描
- 合理使用缓存，减少数据库访问
- 定期清理数据库日志和碎片

## 8. 故障排查

### 8.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|----------|
| 慢SQL监控不生效 | 配置未启用或阈值设置过高 | 检查配置项，调整阈值 |
| 敏感信息未脱敏 | 敏感字段配置不正确 | 检查敏感字段配置 |
| MDC监控不生效 | MDC键配置不正确 | 检查MDC键配置 |
| 性能下降 | 线程池配置不合理或缓存过大 | 调整线程池和缓存配置 |
| 内存泄漏 | 缓存未正确过期或线程池未正确关闭 | 检查缓存配置和资源释放 |

## 9. 版本管理

### 9.1 版本号规则

采用语义化版本管理：

- **主版本号**：不兼容的API变更
- **次版本号**：向后兼容的功能添加
- **修订号**：向后兼容的问题修复

### 9.2 发布流程

1. **开发阶段**：使用SNAPSHOT版本
2. **测试阶段**：使用RC版本
3. **发布阶段**：使用正式版本

### 9.3 升级指南

| 版本 | 升级说明 | 注意事项 |
|------|---------|----------|
| 2.x → 3.x | Spring Boot 2 → Spring Boot 3 | 需要Java 17+，检查依赖兼容性 |
| 3.0.0 → 3.1.0 | 新增功能 | 向后兼容，直接升级 |
| 3.1.0 → 3.2.0 | 性能优化 | 向后兼容，直接升级 |

## 10. 安全建议

### 10.1 配置安全

- 避免在配置文件中硬编码敏感信息
- 使用环境变量或配置中心管理敏感配置
- 定期更新配置，避免使用默认密码

### 10.2 日志安全

- 确保敏感信息已脱敏
- 避免在日志中输出完整的SQL语句和参数
- 定期清理日志，避免日志文件过大

### 10.3 依赖安全

- 定期更新依赖版本，修复安全漏洞
- 使用依赖扫描工具检查安全漏洞
- 避免使用已知存在安全问题的依赖
